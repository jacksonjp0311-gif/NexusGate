# Technology Domain

Purpose: study technology systems, architectures, interfaces, protocols, and operational patterns.

Allowed claims: sourced technology descriptions, reproducible observations, bounded architecture comparisons.

Blocked claims: unsupported vendor claims, security proof without audit, production readiness without gates.

Required evidence: source/provenance, version/date, reproducible command or cited artifact when applicable.

Safe study modes: architecture maps, interface comparisons, risk notes, orchestration plans.

Handoff surfaces: `reports/nexus_domain_intelligence_report_latest.json`, `domains/technology/`.

Compiler gates: `.\scripts\nexus.ps1 domain`, `.\scripts\nexus.ps1 evolve`.

Claim boundary: technology study is local orientation evidence, not production validation.
