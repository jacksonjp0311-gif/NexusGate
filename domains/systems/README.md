# Systems Domain

Purpose: study orchestration, feedback, constraints, failure modes, control loops, and adaptation patterns.

Allowed claims: bounded systems patterns, sourced models, explicit hypotheses, local architecture mappings.

Blocked claims: universal systems law without evidence, metaphor as fact, safety proof from diagramming.

Required evidence: source/provenance, assumptions, scope, failure modes, validation status.

Safe study modes: loop maps, failure-mode cards, feedback diagrams, orchestration cards.

Handoff surfaces: `reports/nexus_domain_intelligence_report_latest.json`, `domains/systems/`.

Compiler gates: `.\scripts\nexus.ps1 domain`, `.\scripts\nexus.ps1 evolve`.

Claim boundary: systems study is orientation evidence, not universal proof.
