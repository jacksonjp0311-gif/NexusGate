# Chat Intelligence Distribution v0.5.1

The Reflective Neural Assembly can now compress the same lab evidence into handoff packets for multiple reasoning and reflection surfaces:

- Codex
- ChatGPT
- local agents
- PowerShell TUI
- Electron HUD

This is not a new authority layer. It is a distribution layer for repo-native intelligence.

## Purpose

Codex and other chat surfaces should not enter NEXUS cold. They should receive a compact, evidence-linked operational packet that tells them:

1. what the system is,
2. what to read first,
3. what evidence exists,
4. what recommendation is currently active,
5. what remains uncertain,
6. what actions are blocked.

## Distribution Law

```text
Distribute intelligence.
Do not distribute authority.
Optimize rehydration.
Do not bypass evidence.
Compress context.
Do not promote unsupported claims.
```

## Surface Roles

| Surface | Role | Authority |
|---|---|---|
| Codex | repo-aware patch/review agent | recommendation only until human-authorized patch |
| ChatGPT | reasoning and interpretation surface | recommendation only |
| local agent | machine-readable orientation consumer | recommendation only |
| TUI | operator reflection surface | presentation only |
| Electron | HUD reflection surface | presentation only |

## Write Surfaces

The distribution layer may write only inside the isolated lab:

```text
labs/reflective_neural_assembly/reports/neural_intelligence_distribution_latest.json
labs/reflective_neural_assembly/handoffs/*.json
labs/reflective_neural_assembly/handoffs/*.md
```

## Blocked Actions

```text
self_authorize
auto_apply_fix
arbitrary_shell
external_api_write
secret_access
parent_repo_mutation
bypass_evolve
memory_promotion_without_evidence
```

## Claim Boundary

Chat intelligence distribution optimizes rehydration for Codex, ChatGPT, local agents, TUI, and Electron. It does not prove correctness, safety, security, production readiness, model understanding, autonomous authority, or real-world truth.
