# NEXUS GATE v0.4.0 Domain Intelligence Orchestrator

NEXUS GATE can study domains and convert reasoning into durable repo-native intelligence. It does not learn by changing model weights.

The learning path is:

```text
study
extract
map
model
simulate
test
gate
reflect
compress
rehydrate
orchestrate
```

The artifact path is:

```text
reasoning -> notes -> concept cards -> schemas -> tests -> gates -> reports -> lineage -> rehydration
```

## Laws

```text
Repo-native learning is allowed.
Autonomous authority is not.
Cross-domain synthesis is allowed.
Unsupported claims are not.
```

NEXUS GATE may study, synthesize, orchestrate, and evolve. It may not abandon evidence, gates, human authority, or claim boundaries.

## Domain Claim Rules

1. Domain study is evidence-gated.
2. Every claim needs source/provenance or must be marked hypothesis/speculation.
3. Math claims require derivation, proof status, or explicit conjecture status.
4. Physics claims require dimensional consistency and simulation/derivation status.
5. Biology claims require source, safety boundary, and no clinical/medical authority.
6. Code claims require tests, lint/type status when applicable, and reproducible commands.
7. Cross-domain analogies are allowed only as analogies unless validated.
8. Codex may orchestrate repo tasks, but cannot self-authorize.

## Claim Boundary

This system can study domains. It cannot certify truth by itself. It can generate hypotheses. It cannot promote hypotheses into facts without evidence. It can run simulations. It cannot treat simulations as empirical proof. It can write code. It cannot call code production-ready without gates. It can organize biology concepts. It cannot provide medical authority or unsafe wet-lab instructions.
