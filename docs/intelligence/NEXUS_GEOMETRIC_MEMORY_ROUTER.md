# NEXUS Geometric Memory Router v0.8.3A

## Purpose

The Geometric Memory Router is the first contract layer that turns the uploaded Codex theory stack into NexusGate operating geometry.

It does not train Mistral.
It does not mutate raw model weights.
It does not grant autonomous authority.
It defines how local model recommendations, repo evidence, human intent, and memory context are projected before bounded repair.

## Source Theory Extraction

```text
LFTE  -> typed depth / projection / substrate geometry
EIMT  -> episodic memory kernel / drift gate / source fallback
RCMA  -> cross-modal adaptation / shared latent remapping
TRAT  -> simplex attractor / bounded persistence / descriptor governance
```

## Four-Axis Tesseract Router

```text
Axis 1: Intent     -> what is being asked?
Axis 2: Evidence   -> what does repo/test/compiler prove?
Axis 3: Authority  -> who may mutate what?
Axis 4: Context    -> which docs/files/memory slices are relevant?
```

A request is not router-ready until all four axes are populated.

## Projection Flow

```text
human intent
-> LFTE typed-depth projection
-> EIMT episodic drift/source gate
-> RCMA cross-modal latent remap
-> TRAT attractor/persistence score
-> NEXUS authority/evidence gate
-> recommend | ask | block | bounded repair
```

## Model Boundary

Mistral is a local reflective organ inside the system. It may recommend, summarize, classify, and compare. It may not self-authorize mutation.

Recommended tuning order:

```text
routing weights
context weights
prompt geometry
role assignment weights
retrieval/memory slice weights
runtime options
LoRA/fine-tuning only after evidence
```

## Claim Boundary

This is a software governance contract, not proof of consciousness, physics, biological mechanism, alignment, production readiness, or model correctness.
