# NEXUS GATE v0.3.7 Reflective Intelligence Loop

NEXUS GATE is a reflective intelligence gateway: reasoning is allowed to move through durable evidence surfaces instead of staying trapped inside a single chat session.

## Meaning

Reflective intelligence in NEXUS means that a model, operator, repo, shell, compiler, TUI, and Electron surface can participate in one governed loop:

```text
hydrate
orient
reason
propose
patch
execute
validate
reflect
compress
resume
```

The model supplies intelligence. The repo preserves intelligence. The gates discipline intelligence. The reflection surface returns intelligence to the next session.

Core law:

```text
Reflective intelligence is permitted.
Autonomous authority is not.
```

The loop may adapt, improvise, and overcome. The loop may not bypass evidence, gates, or human command.

## Surfaces

| Surface | Role |
|---|---|
| ChatGPT / Codex | reasoning surface |
| NEXUS repo | memory/evidence surface |
| PowerShell | execution surface |
| TUI / Electron | reflection surface |
| Human operator | authority surface |

The intelligence is captured in the loop, not only inside the model. A later AI session can rehydrate from current reports and compressed handoff files instead of relying on a full prior transcript.

## ChatGPT Without Direct Repo Access

ChatGPT can operate NEXUS by reading compressed handoff material copied by the operator:

```text
reports/tui/nexus_tui_ai_handoff_latest.txt
reports/tui/nexus_tui_snapshot_latest.html
state/ai_feedback_context_latest.json
docs/feedback/FEEDBACK_LOG.md
reports/nexus_reflective_loop_report_latest.json
state/nexus_lineage_manifest_latest.json
```

ChatGPT may reason, identify risks, draft bounded recommendations, and prepare patches for a human or repo-connected agent to apply. It does not gain direct repo mutation authority.

## Codex With Direct Repo Access

Codex can hydrate directly from the repo, inspect source, apply bounded patches, run tests, run NEXUS lanes, and write evidence. Codex still acts inside the same authority boundary:

```text
No autonomous self-authorization.
No secret access.
No external API writes.
No memory promotion without evidence.
No gate bypass.
```

## TUI And Electron Reflection

The PowerShell HUD TUI and Electron HUD runtime reflect operational truth from NEXUS evidence surfaces. They may make governed actions visible and selectable, but they do not own the logic and do not become the operator.

Electron remains presentation-only. It reads allowlisted evidence surfaces and requests allowlisted lanes through the governed bridge.

## Token Optimization

NEXUS optimizes token usage by compressing current operational truth into durable handoff surfaces:

```text
state/ai_feedback_context_latest.json
docs/feedback/FEEDBACK_LOG.md
reports/nexus_feedback_interface_report_latest.json
reports/nexus_self_healing_report_latest.json
reports/tui/nexus_tui_ai_handoff_latest.txt
reports/tui/nexus_tui_snapshot_latest.html
reports/tui/nexus_tui_surface_latest.json
reports/nexus_electron_preflight_report_latest.json
reports/nexus_electron_smoke_report_latest.json
reports/nexus_reflective_loop_report_latest.json
state/nexus_lineage_manifest_latest.json
state/interface_adapter_contract_index.v0.3.7.json
```

An AI should not need full repo memory every time. It should rehydrate from these compressed surfaces first, then inspect source only when needed.

## Human Authority

Human authorization remains the authority boundary. The reflective loop can propose, route, and validate; it cannot decide to mutate the repo by itself.

## Evidence, Not Proof

Passing gates is evidence. It is not proof of correctness, security, safety, production readiness, model understanding, or autonomous authority.

Final design law:

```text
NEXUS GATE may adapt, improvise, and overcome.
It may not abandon evidence, authority, human command, or gates.
```
