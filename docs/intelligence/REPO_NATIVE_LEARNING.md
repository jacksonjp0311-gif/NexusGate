# NEXUS GATE v0.4.0 Repo-Native Learning

Repo-native learning is persistent improvement of repository structures through evidence, not mutation of model weights.

NEXUS learns when reasoning becomes durable, reviewable, testable, and rehydratable.

## Core Learning Units

```text
SourceCard
ConceptCard
ClaimCard
EquationCard
SimulationCard
CodePatternCard
ExperimentCard
FailureCard
ReflectionCard
OrchestrationCard
```

## Learning Chain

```text
source -> concept -> claim -> model -> test -> report -> lineage -> handoff -> next session
```

## Rules

- A card is not a proof.
- A report is not production readiness.
- A simulation is not empirical validation.
- A source summary is not authority unless provenance is preserved.
- A hypothesis must remain marked as hypothesis until evidence promotes it.
- A repo change must pass gates before it becomes durable system memory.

## Boundary

Repo-native learning improves orientation and compounding discipline. It does not prove scientific truth, mathematical truth, code correctness, safety, security, production readiness, model understanding, or autonomous authority.
