# NEXUS GATE v0.4.0 Cross-Domain Synthesis Protocol

NEXUS can connect biology, physics, math, code, systems theory, and human-AI workflows when the connection is explicitly bounded.

## Rules

```text
No metaphor becomes fact without evidence.
No simulation becomes real-world proof.
No code demo becomes production validation.
No biological pattern becomes medical advice.
No mathematical conjecture becomes theorem without proof.
No physics analogy becomes physical law without dimensional and empirical support.
```

## Allowed Uses

- Analogies for brainstorming.
- Hypotheses with explicit uncertainty.
- Concept maps with provenance.
- Simulations with stated assumptions.
- Code prototypes with tests and non-production boundary.
- Systems patterns with failure modes and limits.

## Required Boundary

Every synthesis must say whether it is:

```text
analogy
hypothesis
validated claim
derived result
simulation result
tested code pattern
speculation
```

## Claim Boundary

Cross-domain synthesis is orientation and hypothesis support. It does not certify truth across domains.
