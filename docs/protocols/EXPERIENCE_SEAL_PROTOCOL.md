# Experience Seal Protocol

An Experience Seal is a compact, auditable object for a completed governed action.

Inputs:

- recommendation receipt;
- authorization receipt;
- execution receipt;
- effect receipt;
- action-bound final evolve receipt;
- validation receipt;
- learning receipt;
- semantic action-chain verification.

The experience ID is derived from the source epoch, action ID, and verified receipt references.

Experience Seals can inform future routing. They do not grant authority and do not apply calibration by themselves.
