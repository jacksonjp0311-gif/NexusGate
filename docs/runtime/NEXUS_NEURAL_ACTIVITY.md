# Nexus Neural Activity Integration

Version: v0.1.0

Purpose: port the Neural Cathedral / Bioelectric Engine HTML surface into NexusGate as a governed visual module.

## Surfaces

```text
neural_activity/index.html
electron/renderer/nexus_neural_activity_panel.v0.1.0.css
electron/renderer/nexus_neural_activity_bridge.v0.1.0.js
docs/runtime/NEXUS_NEURAL_ACTIVITY.md
state/neural_activity/neural_activity_manifest.v0.1.0.json
tests/test_neural_activity_integration.py
```

## HUD Contract

The Electron HUD gets a `neural-activity-panel` inside the left rail. The panel embeds a light preview with:

```html
<iframe src="../../neural_activity/index.html?embed=1"></iframe>
```

The full program opens through a pop-out button that loads:

```text
../../neural_activity/index.html
```

## Desktop Portal Contract

The desktop entry portal gets:

```text
[13] Neural Activity / Cathedral -> bioelectric popout HUD
```

## Boundary

This module is a visual surface only. It does not change authority, governance, NexusCell execution, model routing, or memory policy.

## CLOSE V Absolute Layout Fix

Close U still showed the preview compressed into the top strip. The actual failure was layout ownership: the mounted content existed, but it did not claim the body area.

Close V uses a final layout strategy:

```text
- cache-busted V bridge/CSS assets
- runtime JS claims the reserved box
- host becomes position: relative
- header is absolute at top
- body is absolute from top:46px to bottom:8px
- native canvas fills body
```

This avoids grid/flex collapse from older HUD CSS.
