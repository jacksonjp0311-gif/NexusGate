# Nexus Failure Intelligence Distributor

Version: v0.1.3

Purpose: convert each failed validation run into reusable local intelligence that a human or rehydrating AI can load before writing the next close script.

This is not autonomous authority. It is an evidence distribution layer.

## Distribution Law

```text
failure
-> compact wound
-> verifier
-> stability lock
-> routed lesson
-> rehydration-visible state
-> next close target
```

## What gets distributed

```text
reports/nexus_compiled_failure_latest.md
reports/nexus_compiled_failure_latest.json
state/failure_intelligence/nexus_failure_intelligence_latest.json
state/failure_intelligence/nexus_failure_intelligence_manifest.v0.1.3.json
docs/runtime/NEXUS_FAILURE_INTELLIGENCE_DISTRIBUTOR.md
```

## Rehydration rule

Before a new repair script, inspect the latest failure intelligence packet. If it contradicts chat memory, the packet wins.
